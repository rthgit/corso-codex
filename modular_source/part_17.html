Accuracy and Sources Audit

What is officially documented, what is an advanced pattern, and what to verify before use.

Features treated as documented

  • Codex App as command center: parallel threads, worktrees, automations, and Git functionality are described in the Codex App documentation.
  • Windows App: Windows documentation indicates support for worktrees, automations, Git functionality, in-app browser, artifact previews, plugins, and skills.
  • Worktrees: documented as a method for running independent tasks in the same project without interference.
  • Automations: documented with local mode and dedicated background worktrees for Git repositories.

Advanced educational patterns, not official promises

  • Worker role cards: architectural pattern for orchestrating specialized agents.
  • .codex/agents/ local state: useful pattern for advanced teams and courses, to be adapted to the project.
  • Orchestrator + Vector DB: long memory pattern; should not be presented as an automatic Desktop App feature.
  • Fake vector DB JSON: educational exercise to simulate memory retrieval and governance.

To be verified at the time of use

  • Computer Use: may vary by platform, geographic area, plan, and rollout.
  • Specific Plugins: availability and names may change.
  • Recommended Models: do not stall the course on unverified model names or costs.
  • Local Commands: adapt build/test to each repository; CMake examples are templates.
Course Editorial Rule: when something is an OpenAI feature, present it as an app function. When it's an agentic architecture practice, present it as an advanced pattern. Do not mix the two categories.
