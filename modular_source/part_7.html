Automations — codex-action and CI/CD

GitHub Actions with openai/codex-action, schedule, Triage Inbox, and full pipelines

OpenAI officially publishes openai/codex-action (github.com/openai/codex-action): a GitHub Action that runs Codex within your CI/CD workflows. No servers to manage, no external cron jobs.

⚡
Quick Setup: Go to Settings → Secrets and variables → Actions in your repo and add OPENAI_API_KEY as a secret. Then copy the YAML below.

Workflow 1 — Automatic PR Review on every Pull Request

# .github/workflows/codex-pr-review.yml
name: Codex PR Review

on:
  pull_request:
    types: [opened, synchronize, reopened]

jobs:
  codex-review:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
        with:
          ref: refs/pull/${{ github.event.pull_request.number }}/merge

      - name: Run Codex Review
        id: codex
        uses: openai/codex-action@v1
        with:
          openai-api-key: ${{ secrets.OPENAI_API_KEY }}
          prompt: |
            You are a senior Staff Engineer. Analyze the changes in this PR.
            Look for: logic bugs, security issues (injection, auth bypass),
            lack of error handling, fragile assumptions.
            Output format: CRITICAL / WARNING / SUGGESTION / OK.
          sandbox: read-only

      - name: Comment on PR
        uses: actions/github-script@v7
        with:
          script: |
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: `## 🤖 Codex Review\n\n${{ steps.codex.outputs.final-message }}`
            })

Workflow 2 — Daily Security Scan with Auto-Fix

# .github/workflows/codex-security-scan.yml
name: Security Scan + Auto-Fix

on:
  schedule:
    - cron: '0 6 * * *'  # Every day at 06:00 UTC
  workflow_dispatch:     # Allow manual execution

jobs:
  security-fix:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - run: npm ci

      - name: Codex Security Fix
        uses: openai/codex-action@v1
        with:
          openai-api-key: ${{ secrets.OPENAI_API_KEY }}
          prompt: |
            1. Run `npm audit` and read the output
            2. For each critical or high CVE, update the dependency
            3. Run the tests: `npm test`
            4. If tests pass, create a branch fix/security-$(date)
               with the changes and open a PR
            5. If tests fail, document in SECURITY-REPORT.md
               the unapplicable fixes and the reason
          sandbox: workspace-write
