Skills and Plugins — Reusable Custom Capabilities

Create, install, and distribute skills company-wide

A Skill is a package of instructions, resources, and optional scripts that teaches Codex how to execute a specific workflow. It's the way to institutionalize your team's processes and make them agentic.

You can share an entire .agents/skills directory among multiple developers, ensuring that the agent executes standard company procedures (e.g., release process, DB migration).

FilePurpose
SKILL.mdMain natural language instructions. Use frontmatter for name/description and triggers.
agents/openai.yamlMetadata, security policies (which commands it can execute), tool deps, and models.
scripts/Optional scripts executed by the skill (e.g., pre-flight.sh or post-review.py).
resources/Reference documents, PR examples, JSON or Markdown file templates.
 
.agents/skills/
└── release-manager/
    ├── SKILL.md                 # How to generate changelog and bump version
    ├── agents/
    │   └── openai.yaml          # Permission for `git tag` and `npm publish`
    └── scripts/
        └── verify-tests.sh      # Executed automatically before release
