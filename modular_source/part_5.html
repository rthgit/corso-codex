Model Context Protocol (MCP)

Connecting Codex to GitHub, Linear, Slack, DB, and lazy loading

The Model Context Protocol is the de facto standard of 2026 for connecting Codex to external data sources (Notion, Figma, SQL databases) without copying/pasting text.

⚡
Lazy Loading: MCP servers insert a lot of information into the Codex Context Window. Use ENABLE_TOOL_SEARCH="true" to load MCP tools dynamically, saving 30% of tokens.
# Connection configuration in ~/.codex/config.toml
[[mcp_servers]]
name = "github"
command = "npx"
args = ["-y", "@modelcontextprotocol/server-github"]
env = { GITHUB_PERSONAL_ACCESS_TOKEN = "$GITHUB_PAT" }

[[mcp_servers]]
name = "postgres"
command = "npx"
args = ["-y", "@modelcontextprotocol/server-postgres", "postgresql://user:pass@localhost/db"]

Once configured, you can ask Codex: "Read issue #42 on GitHub, query the postgres database to find the offending user, and implement the fix."
