AGENTS.md — Persistent Project Instructions

Governance, conventions, test commands, and "Gotchas"

AGENTS.md (also known as CLAUDE.md or system rules) is a text file, similar to README.md, that Codex reads automatically at the beginning of each session in your repository.

ℹ
2026 Best Practice: Keep AGENTS.md under 200 lines. Use explicit paths and "Context Freshness" rules to prevent the agent from hallucinating non-existent patterns.

2.1 How Codex Discovers AGENTS.md — The Official Hierarchical System

Codex doesn't read just one AGENTS.md: it builds a hierarchical instruction chain by concatenating multiple files, from the repo root down to your current directory. Rules closer to the CWD take priority over global rules.

Discovery OrderChecked FileBehavior
1. Global$CODEX_HOME/AGENTS.override.mdIf it exists, overwrites everything. Company policy file, non-modifiable by devs.
2. Global fallback$CODEX_HOME/AGENTS.mdPersonal global rules (e.g., preferred style, default tools).
3. Project root[git-root]/AGENTS.mdStandard project rules, read by the whole team.
4. SubdirectoriesEvery AGENTS.md in intermediate pathsRules for submodules. Concatenated in root→CWD order.
5. CWD[cwd]/AGENTS.md or AGENTS.override.mdHighest priority. Local override for isolated dev or testing.
ℹ
Total Limit: The process stops when the sum of the files reaches project_doc_max_bytes (default 32 KiB, configurable in config.toml). Files not included are silently ignored.

2.2 Advanced AGENTS.md Configuration in config.toml

# ~/.codex/config.toml

# Increase limit if AGENTS.md gets truncated (check with `codex status`)
project_doc_max_bytes = 65536  # Default: 32768 (32 KiB)

# Alternative filenames if AGENTS.md does not exist (custom naming monorepos)
project_doc_fallback_filenames = ["TEAM_GUIDE.md", ".agents.md", "CODEX.md"]

2.3 Full AGENTS.md for Node.js/TypeScript Project

# AGENTS.md

## Project
This is a REST API in Node.js + TypeScript (Express).

## Architecture and Naming
- src/routes/ → Express endpoints. Always use `kebab-case` for route files.
- src/services/ → business logic. Use classes with dependency injection.
- src/models/ → Prisma schemas.

## Essential Commands
- Build: `npm run build`
- Test: `npm test` (DO NOT modify Jest config files)
- Lint: `npm run lint -- --fix`

## Security Rules
- NEVER hardcode API keys in the code. Use dotenv and `process.env`.
- Always verify user permissions in controllers, not in services.

## "Gotchas" (Decision Log)
- [Common Bug]: When adding a new Prisma model, remember to update the seed file `prisma/seed.ts`.
- [Tip]: Authentication is handled by the `authMw` middleware, do not apply it manually to public routes.

## Context Freshness
Before starting modifications, always run `git status` and re-read the latest PRD.
