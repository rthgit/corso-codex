Codex CLI — The Enhanced Terminal

Installation, configuration, advanced commands, non-interactive exec, scripting, and context management

1.1 Installation and Setup

The Codex CLI is open source, written in Rust to guarantee sub-millisecond latencies in file system management, and is available on macOS, Windows, and Linux. It integrates a local tokenizer for exact real-time context calculation.

# Installation via npm (requires Node 18+)
npm install -g @openai/codex

# First launch and interactive authentication
codex

# Headless authentication (for CI/CD)
export OPENAI_API_KEY="sk-..."
codex login --headless

# Check status, active model, and remaining tokens
codex status

1.2 The 3 Official Approval Modes — The Most Important Choice

OpenAI officially documents three autonomy levels for the Codex CLI. The wrong choice causes either continuous blocks (suggest mode on CI pipelines) or security risks (full-auto on production repositories).

ModeSandboxApproval PolicyWhen to Use It
Suggest (default)read-onlyCodex shows proposed changes, you approve every single actionExploration, code review, first use on an unknown repository
Auto Editworkspace-writeModifies files autonomously, but asks for confirmation for shell commandsDaily active development. The optimal compromise for most devs
Full Autoworkspace-writeExecutes everything autonomously (on-request): code + shell + commandsCI/CD, isolated Docker containers, nightly pipelines. NEVER on production repositories without git checkpoints
 
# Launch with explicit approval mode
codex --approval-policy suggest        # Safe mode (default)
codex --approval-policy auto-edit      # Modifies files, asks for shell
codex --full-auto                      # Alias for full auto (on CI/CD)

# Equivalent permanently configured in config.toml
[core]
approval_policy = "auto-edit"

1.3 Non-Interactive Mode — exec

Transforms Codex into an automated CLI tool, ideal for pre-commit hooks, CI/CD pipelines, or cron jobs. The --max-budget-usd flag is the only guardrail against infinite loops or anomalous billing.

# Execution with budget cap (prevents surprise billing)
codex exec --max-budget-usd 0.50 "Generate unit tests for src/auth.ts"

# JSON output for structured parsing in bash
codex exec --json "Analyze the code and return bugs" | jq '.bugs[]'

# Full auto on cloud container (safe because isolated)
codex cloud exec --env ENV_ID --full-auto --attempts 3 "Fix bug #123"

# Pipeline pre-commit hook (example .husky/pre-commit)
codex exec --approval-policy auto-edit --max-budget-usd 0.10 "Fix lint errors"

1.3 Context Management

Codex's "Context Window" is not infinite. A "polluted" context leads to buggy code. Senior developers use these commands to keep the context "fresh".

CommandDescription and When to Use It
/compact [focus]Compresses the current session preserving technical decisions. Use e.g. /compact focus on auth rules when you exceed 70% of the context.
/clearClears the context. Always use before starting a completely unrelated new task to prevent pollution.
/rewind (or Esc+Esc)Undoes the last N actions and returns to a clean checkpoint if Codex has taken a wrong direction.

1.4 Navigation and Shortcuts

CommandDescription
/modelChanges model during a session (e.g., from 5.5 to plan to 5.4-mini to write boilerplate)
/reviewStarts automatic review of uncommitted changes
@Fuzzy search to force inclusion of a specific file in the context (much more token-efficient than a verbal query)
!Executes shell commands and injects output into the context (e.g., !npm run test)
/costShows current spend in USD and tokens of the session

1.5 config.toml Configuration

The global Codex configuration, which defines security constraints (sandbox) and approval policies for destructive commands.

# ~/.codex/config.toml

[core]
model = "gpt-5.4"
# Strict policy: Codex asks permission for everything except allow-listed commands
approval_policy = "unless-allow-listed" 

[sandbox]
# "workspace-write" allows changes only inside the current git folder
mode = "workspace-write"

[[mcp_servers]]
name = "github"
url = "https://api.githubcopilot.com/mcp/"

[[rules.allow]]
command = "git"
args = ["status", "diff"] # Codex can execute these without confirmation

[agents]
max_concurrent = 4
budget_cap_session_usd = 2.0
