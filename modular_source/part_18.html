Cheat Sheet — Complete Quick Reference

CLI — Essential Commands

npm install -g @openai/codex             # Installation (requires Node 18+)
codex                                     # Start interactive session (Suggest mode)
codex --model codex-mini-latest "Prompt"  # Lightweight model for simple tasks
codex --model codex-1 "Prompt"            # Advanced model for complex tasks
codex --approval-policy suggest           # Shows changes, you approve everything
codex --approval-policy auto-edit         # Modifies files, asks only for shell
codex --full-auto                         # Fully automatic (CI/sandbox only)
codex exec "Task"                         # Non-interactive execution
codex exec --json "Task" | jq             # Parsable JSON output
codex exec --max-budget-usd 0.50 "Task"   # With security budget cap
codex --worktree feature-name "Task"      # Agent on isolated Git worktree
codex mcp list                            # List connected MCP servers
codex status                              # Active model, used tokens, cost

Session Slash Commands

/model codex-mini-latest     # Switch to budget model
/model codex-1               # Switch to advanced model
/compact focus on auth        # Compact preserving context on the topic
/clear                        # Reset context (new unrelated task)
/rewind                       # Back to previous checkpoint (undo actions)
/review                       # Automatic review of uncommitted diffs
/skills                       # List available skills in the project
/cost                         # Current session spend in USD + tokens
@src/auth/login.ts            # Force file inclusion in context (token-efficient)
!npm test                     # Run shell and inject output into context

Approval Modes — Decision Table

suggest     → sandbox: read-only   → approve everything: ideal for exploration
auto-edit   → sandbox: workspace-write → modifies files, asks for shell: daily development
full-auto   → sandbox: workspace-write → fully automatic: only CI/CD and isolated containers

codex-action — GitHub Actions

# Minimal setup in .github/workflows/codex.yml
- uses: openai/codex-action@v1
  with:
    openai-api-key: ${{ secrets.OPENAI_API_KEY }}
    prompt: "Your task here"
    sandbox: read-only  # or: workspace-write

Models — When to Use Them

codex-mini-latest  → bug fix, test, boilerplate (fast, cheap)
codex-1            → refactoring, multi-file feature (high quality)
gpt-5.5            → architectural planning, Computer Use

Environment Variables

OPENAI_API_KEY=sk-...         # API authentication (mandatory for headless)
CODEX_HOME=$(pwd)/.codex      # Project-specific config

Configuration Files

~/.codex/config.toml     # Global config (model, approval_policy, sandbox)
.agents/skills/          # Repository skills (shared with team)
AGENTS.md                # Persistent instructions (max 200 lines, golden rule)
HANDOFF.md               # Context across sessions (generate before closing)
~/.codex/log/            # Audit trail: every action, tokens, cost
