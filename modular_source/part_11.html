Codex Enterprise — Governance and Security

Admin setup, managed config, permission profiles, and audit trail

In corporate environments, Codex must operate within precise and trackable boundaries. The Enterprise tier adds centralized governance.

Pillars of Enterprise Security

PillarImplementation
Config ManagedA centralized config.toml file distributed via Git or a corporate secret manager (e.g., AWS SSM). Developers cannot override deny policies.
Permission ProfilesPre-defined profiles by role (e.g., junior-dev: read-only and tests; senior-dev: write with approval; ci-bot: non-interactive exec with budget cap).
Global AGENTS.mdAn AGENTS.md in the corporate root (read by all repos via Git submodule) enforces uniform code standards and prohibits insecure patterns.
Audit LoggingEvery tool call, modified file, and executed command is saved in ~/.codex/log/ with a timestamp, token cost, and output hash. Integrable with corporate SIEMs.
Zero-Trust Sandboxworkspace-read mode: Codex can read but not modify files outside of explicitly permitted directories. Ideal for code review agents.
 
# Example: minimal enterprise config
[core]
approval_policy = "always"  # Every action requires approval

[sandbox]
mode = "workspace-read"  # Ultra-secure default, overridable per project

[audit]
enabled = true
export_path = "/var/log/codex-audit/"  # Centralized path for SIEM
