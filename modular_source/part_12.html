Advanced Prompt Engineering for Codex

RIPER Framework, 5 Principles, self-healing, and master techniques

The 5 Codex Prompt Principles

  • Task context: what needs to be done, why, and what is the measurable success metric.
  • Verifiable evidence: specify the tests to run, the files that must be created, or the commands whose output proves success.
  • Explicit constraints: explicitly state what NOT to touch (e.g., "Do not modify the database schema").
  • Output format: structured result (JSON, Markdown, diff) for automated pipelines.
  • Iteration: incremental and verifiable tasks at each step, not a monolith to validate at the end.

RIPER Framework (Research → Innovate → Plan → Execute → Review)

The definitive structured workflow for highly complex tasks. Enforces a strict separation between phases: Codex cannot "jump" to implementation before understanding the problem.

PhaseInstruction to CodexExpected Output
Research"Read the codebase. Do not write anything. Just reply to me with what you understood."Code map, dependencies, existing patterns.
Innovate"Propose 3 different architectural approaches for this problem with pros and cons."Architectural options, no code.
Plan"Write a step-by-step plan. For each step, indicate the file, the test, and the success criteria."Detailed, approvable plan.
Execute"Implement the approved plan. One step at a time. Stop after each step and wait for my confirmation."Incremental diffs, passing tests.
Review"Use an empty context subagent to perform an adversarial review of the implementation."CRITICAL/WARNING/SUGGESTION report.
