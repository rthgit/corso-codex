Subagents — Parallel Multi-Agent Orchestration

Specialized agents in parallel, Test Time Compute, and advanced patterns

Codex's multi-agent architecture uses isolated Git worktrees as sandboxes for each parallel agent. Each agent operates on a separate branch, without interfering with the others. The coordinator (main agent) integrates the results at the end.

The 3 Fundamental Subagent Patterns

PatternHow it worksIdeal use case
Independent Parallel3-4 agents on different worktrees work simultaneously on unrelated featuresSprints with multiple independent tickets, maximum velocity
Compete & Pick2-3 agents implement the same feature differently; a reviewer chooses the best oneCritical architectural decisions, risky refactoring
Sequential PipelineArchitect → Developer → Tester → Reviewer, each with a clean contextComplex features requiring enterprise quality

Parallel Agents Configuration in config.toml

# ~/.codex/config.toml
[agents]
max_concurrent = 4              # Maximum 4 parallel agents
model = "codex-mini-latest"     # Lightweight model for subagents
worktree_base = "/tmp/codex-worktrees"  # Base directory for worktrees

Pattern: Test Time Compute (Adversarial Reviewer)

A fresh context window finds errors that the person who wrote the code cannot see. It's the most effective pattern for increasing quality without additional human cost.

🎯 PROMPT — Adversarial Reviewer with Isolated Subagent
PHASE 1 (Main Agent): Implement feature X. Upon completion, write the git diff to /tmp/diff-for-review.patch. PHASE 2 (Subagent with empty context): Read only the file /tmp/diff-for-review.patch. You have no context of how it was written. Act as a Staff Engineer with 15 years of experience. Critically analyze: - Memory leaks or resource leaks - Race conditions - Unsanitized inputs (injection) - Faulty logic at edge cases - Missing or insufficient tests Mandatory output: CRITICAL: [severe problem blocking merge] WARNING: [minor problem to fix before merge] SUGGESTION: [optional improvement] OK: [correct aspects] PHASE 3 (Main Agent): Fix all CRITICAL issues before replying to me.
